# Product Overview

## Product Name
The Dispatch Company

## One-Line Description
The Dispatch Company is an operating system for boutique hotel operations that turns daily chaos into clear, actionable work.

## Problem
Independent and boutique hotel operators often run properties through:
- memory
- texting
- paper
- fragmented PMS notes
- reactive decision-making

This forces owners and managers to spend too much time inside the building and creates weak delegation, poor accountability, and inconsistent execution.

## Product Thesis
This product helps management:
- set the day
- route issues clearly
- assign work
- enforce quality
- preserve institutional knowledge
- surface management actions automatically

## Core Product Philosophy
- Reduce noise
- Make work clear
- Keep the interface calm
- Replace reports with action
- Replace verbal chaos with structured execution
- Build for how operators actually run properties

## Target Users
### Primary
- boutique hotels
- independent inns
- repositioned motels
- micro-resorts
- experiential lodging operators
- 15 to 60 room properties

### Initial Beta User
Jennifer and her current property

## Core System Model
- Dispatch = context
- Tasks = execution
- Activity = signal
- Staff = accountability and memory
- Admin Cards = system-generated management action

## Key Differentiator
This is not a PMS, not chat, and not a project management tool.
It is a day-to-day property operations system.

## Future Direction
Over time, the system can become:
- a structured operational memory layer
- a trigger engine for cards
- a lightweight intelligence layer that turns notes and events into work

But the MVP is not an AI product. It is a clean execution system first.
